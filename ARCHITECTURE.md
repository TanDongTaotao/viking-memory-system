# Viking 记忆系统 - 架构设计

## 系统概述

Viking 是一个分布式多 Agent 记忆管理系统，采用五层时间衰减机制实现智能记忆管理。

```
┌─────────────────────────────────────────────────────────────┐
│                      Viking Memory                          │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐         │
│  │   Loader    │  │   Saver     │  │  Searcher   │         │
│  │ (autoload)  │  │  (save)     │  │  (recall)   │         │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘         │
│         │                │                │                 │
│  ┌──────┴────────────────┴────────────────┴──────┐         │
│  │              Viking Workspace                 │         │
│  │  ┌────────┬────────┬────────┬────────┐    │         │
│  │  │  L1    │  L2    │  L3    │  L4/L5 │    │         │  
│  │  │ Config │  Hot   │  Warm  │  Cold  │    │         │
│  │  └────────┴────────┴────────┴────────┘    │         │
│  └──────────────────────────────────────────────┘         │
│                          │                                 │
│  ┌───────────────────────┴────────────────────────┐      │
  │              Embedding + Filter                  │      │
  │  (viking-embed.py + sensitive_filter.py)        │      │
  └──────────────────────────────────────────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

## 核心组件

### 1. 记忆加载器 (sv_autoload.sh)

- 会话开始时触发
- 加载优先级: config > hot > todo
- 限制输出 tokens (2k 以内)

### 2. 记忆保存器 (sv_save.sh)

- 会话结束时触发
- 自动生成每日日志
- 支持 sv_write 和 memory-pipeline 两种方式

### 3. 记忆搜索器 (sv_recall.sh)

- 关键词全文搜索
- 支持正则表达式
- 输出匹配上下文

### 4. 权重计算器 (sv_weight.sh)

- 根据访问时间计算权重
- 实现五层衰减逻辑

### 5. 记忆压缩器 (sv_compress.sh)

- 归档 90 天+ 冷数据
- 释放存储空间
- 可选 gzip 压缩

### 6. 实时监听器 (sv_memory_watcher.sh)

- inotify 监控文件变化
- 自动同步修改
- 后台守护进程

## 五层衰减机制

```
时间 ─────────────────────────────────────────────────►

权重:  1.0    0.8    0.5    0.2    0.1    0
       │      │      │      │      │      │
层级:  L1     L2     L3     L4     L5     归档
       │      │      │      │      │      │
名称:  核心   热     温     冷     冻结   压缩
       │      │      │      │      │      │
文件: config  hot/   daily/ long-  .arch- ive/
       .md    7days  30days term   90days+ .gz
```

### 权重计算公式

```
weight = base_weight * access_factor * decay_factor

其中:
- base_weight: 基础权重 (0.1-1.0)
- access_factor: 访问频率因子 (1.0-1.5)
- decay_factor: e^(-λ * days), λ = 0.05
```

## 扩展点

### 1. 自定义向量模型

```python
# 修改 viking-embed.py
model = SentenceTransformer("your-model")
```

### 2. 自定义过滤规则

编辑 `sensitive_filter.py` 的 FILTER_PATTERNS

### 3. 钩子扩展

在 `agent-hooks.yaml` 添加新钩子类型

## 性能优化

- 记忆加载限制 2k tokens
- 搜索结果限制 20 条
- 压缩 90 天+ 冷数据
- 向量批量处理
